# Yandex Station Max Flasher

Графическое приложение для Windows для прошивки, дампа и редактирования образов
устройства **Yandex Station Max** на чипе **Amlogic S905X2 (g12a)**.

Программа объединяет в одном окне прошивку через `update.exe`, дамп разделов,
редактор переменных окружения U-Boot (ENV), редактор образов Android (через MIK)
и встроенный COM-терминал (UART) для мониторинга загрузки в реальном времени.

---

## Возможности
 
- **Прошивка разделов** — выбор образов и запись на устройство через загрузку
  временного U-Boot (`pyamlboot`) и `update.exe`. Прогресс показывается в
  процентах. Опционально — очистка `data`/`cache` (сброс к заводским).
- **Редактор ENV** — чтение переменных окружения U-Boot с устройства, их правка
  с **корректным пересчётом CRC32** при бинарной записи. Поддержка разблокировки
  bootloader (`lock`, `avb2`, `silent`, SELinux) и спуфинга идентификаторов.
- **Дамп разделов** — чтение разделов с устройства в `.img`. Таблица разделов
  Yandex Station Max встроена; поддержка автоопределения через UART и ручной
  вставки лога `amlmmc part 1`.
- **Прошивка burning-пакета** — прошивка целого `aml_upgrade_package.img`
  (формат Amlogic V2), как это делает официальный USB Burning Tool.
- **Сборка burning-пакета** — из шаблона с подменой разделов своими файлами
  собирается новый `aml_upgrade_package.img`.
- **Редактор образов** — распаковка/сборка образов Android (ext4 / sparse /
  erofs) через консольные утилиты MIK. Патч `vbmeta` (disable-verity) для
  лечения dm-verity бутлупа.
- **COM-терминал (UART)** — мониторинг загрузки в реальном времени. Фильтрует
  виртуальные/Bluetooth-порты, показывает только реальные USB-UART.
- **Кастомные разделы** — добавление произвольного раздела со своим образом.
- **Диагностика USB** — поиск устройства Amlogic (`VID_1B8E` `PID_C003`)
  несколькими методами.
- **Автозагрузка зависимостей** — скачивание утилит из GitHub одной кнопкой.

## Требования

- **Windows 10/11**
- **Python 3.7+**
- Python-пакеты:
  ```
  pip install pyusb pyserial
  ```
  `pyserial` нужен для COM-терминала (опционально). `pyusb` — для загрузки U-Boot
  и USB-диагностики.
- **Драйверы Amlogic USB Burning Tool** (или WinUSB через Zadig) — без них
  устройство не определяется. См. раздел «Драйверы» ниже.
- Для редактора `update.exe` может потребоваться **Microsoft Visual C++ 2010
  Redistributable** (обычно уже установлен в Windows).

## Установка и структура папок
 
Распакуйте проект **ближе к корню диска**; путь не должен содержать пробелов и
кириллицы (требование MIK для сборки образов).
 
```
flasher/
├── gui.py                  ← главный файл, запуск отсюда
├── flasher_settings.json   ← настройки (создаётся автоматически)
├── files/                  ← утилиты (скачиваются автоматически)
│   ├── update.exe          ← прошивка/дамп Amlogic
│   ├── aml_bundle.img      ← U-Boot образ для USB-режима (S905X2)
│   ├── libusb-1.0.dll
│   ├── tools/windows/
│   │   └── aml_image_v2_packer.exe   ← упаковщик пакетов (из khadas/utils)
│   └── MIK/                ← Multi Image Kitchen
│       ├── MIK64.exe
│       └── bin/            ← simg2img, img2simg, make_ext4fs, imgextractor …
├── pyamlboot_local/        ← модуль загрузки U-Boot по USB
│   └── boot.py
└── images-bkp/             ← образы разделов для прошивки (готовите сами)
    ├── boot.img
    ├── system.img
    └── …
```

### Автоматическая загрузка файлов
 
При первом запуске программа проверяет наличие `update.exe`, MIK и
`pyamlboot_local/` и предлагает скачать недостающее. Можно вызвать вручную
кнопкой **«📥 Загрузить утилиты»**. Скачивается весь репозиторий
**suddosu/yasta_flasher** одним zip-архивом (гарантированно попадают все файлы),
MIK — отдельным zip из **CryptoNickSoft/MIK**.
 
> ⚠️ Если `pyamlboot_local/` появилась только после скачивания — **перезапустите
> программу**.
 
Файлы для подготовки **вручную**:
- `files/aml_bundle.img` — U-Boot образ устройства;
- образы разделов в `images-bkp/`;
- `aml_image_v2_packer.exe` — для работы с burning-пакетами (кнопка скачивания
  есть в окне «Burning-пакет»; в USB Burning Tool v2 его НЕТ — только
  `AmlImagePack.dll`, которая из консоли не вызывается).

## Драйверы

Без драйверов Amlogic устройство не появится в системе. Варианты установки:

1. **USB Burning Tool** — установите официальную утилиту, она поставит драйверы.
   После можно удалить саму программу, драйверы останутся.
2. **Zadig** (`https://zadig.akeo.ie/`) — Options → List All Devices → выберите
   `GX-CHIP` / `Amlogic` (VID `1B8E`, PID `C003`) → установите драйвер **WinUSB**
   или **libusb-win32**.

В программе есть кнопка **«🔍 Диагностика USB»**, которая показывает, видит ли
система устройство, и подсказывает, что делать, если нет.

## Аппаратное подключение

1. Найдите сервисную USB-колодку под резинкой возле радиатора.
2. **Замкните пин 6 на пин 3 (GND)** для входа в USB Boot режим.
3. Подключите USB к ПК.
4. Подайте питание на устройство.

Для UART-мониторинга подключите USB-UART преобразователь (CH340, CP2102, FT232…)
к пинам TX / RX / GND, скорость **115200 8N1**.

## Как пользоваться
 
### Прошивка
 
1. Подготовьте образы в `images-bkp/` или укажите кнопками «Обзор…».
2. Отметьте разделы галочками.
3. При необходимости включите **«🧹 Очистить data и cache»**.
4. Нажмите **«🚀 Начать прошивку»**.
5. Программа загрузит U-Boot, прочитает ENV и предложит открыть редактор.
   Прошивка начнётся после закрытия редактора.
> **Минимум для разблокировки** (`silent=0`, `lock=10000000`, `avb2=0`)
> применяется ВСЕГДА — даже если редактор закрыт «Отменой». Эти значения
> устанавливаются в активной сессии U-Boot, иначе прошивка их не воспринимает.
 
### Редактор ENV
 
Вкладки:
- **Идентификация / Безопасность / Система** — типовые переменные.
- **📝 Все переменные** — полный список `key=value` для прямого редактирования.
Изменения `serial`/`deviceid`/`aml_serial`/`cmdline_keys` записываются **бинарно**
с пересчётом CRC32 (через `update partition env`), так как `cmdline_keys` — это
U-Boot-скрипт со спецсимволами, который нельзя задать обычным `setenv`.
 
### Дамп разделов
 
1. **«💾 Дамп разделов»** — таблица YSM уже загружена, дамп доступен сразу.
2. **«⚡ Запустить сессию»** — загрузка U-Boot и (с UART) автоопределение таблицы.
3. Отметьте разделы (есть «Выбрать/Снять все»), укажите папку, дампьте.
### Burning-пакеты
 
**Прошивка** (`📦 Burning-пакет`): выберите `aml_upgrade_package.img` →
распакуйте → прошейте. Программа воспроизводит последовательность USB Burning
Tool для S905X2 (DDR init → U-Boot → разделы → reset).
 
**Сборка** (`🔨 Собрать пакет`): загрузите шаблон (распакованный пакет этого
устройства) → дабл-кликом подмените разделы своими файлами → соберите новый
`aml_upgrade_package.img` для оригинального USB Burning Tool.
 
> Нужен `aml_image_v2_packer.exe` из `github.com/khadas/utils`
> (`aml-flash-tool/tools/windows/`). Кнопка скачивания есть в окне.
 
### Редактор образов
 
1. **«📝 Редактор образов»** → выберите образ (формат определяется автоматически).
2. **«📂 Распаковать»** → редактируйте файлы → **«📦 Собрать образ»**.
3. **«🔓 vbmeta disable-verity»** — патчит `vbmeta.img` (флаги AVB) для лечения
   dm-verity бутлупа после прошивки несовместимых разделов.
> ⚠️ Путь к образам не должен содержать пробелов и кириллицы.
 
### COM-терминал
 
Панель справа. Выберите реальный USB-UART порт (виртуальные/Bluetooth
отфильтрованы), подключитесь — увидите вывод U-Boot и загрузки. Прогресс
прошивки отображается корректно (обновление строки, не столбцом).

## Важные предупреждения
 
- **Не отключайте устройство во время прошивки** — риск «кирпича».
- Делайте **резервную копию** разделов перед прошивкой.
- Образы должны соответствовать вашей модели устройства.
- Операции с RPMB/eFuse физически необратимы — программа их намеренно не трогает.
- Используйте на свой страх и риск.

## Используемые инструменты
 
- **update.exe** — Amlogic USB Burning Tool (CLI): прошивка, дамп, U-Boot.
- **pyamlboot** — загрузка временного U-Boot по USB.
- **aml_image_v2_packer** (khadas/utils) — упаковка/распаковка burning-пакетов.
- **MIK** (`github.com/CryptoNickSoft/MIK`) — распаковка/сборка образов Android.
- **pyserial / pyusb** — COM-терминал и USB-диагностика.
