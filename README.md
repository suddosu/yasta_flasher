# Yandex Station Max Flasher

Графическое приложение для Windows для прошивки, дампа и редактирования образов
устройства **Yandex Station Max** на чипе **Amlogic S905X2 (g12a)**.

Программа объединяет в одном окне прошивку через `update.exe`, дамп разделов,
редактор переменных окружения U-Boot (ENV), редактор образов Android (через MIK)
и встроенный COM-терминал (UART) для мониторинга загрузки в реальном времени.

---

## Возможности

- **Прошивка разделов** — выбор нужных образов и запись их на устройство через
  загрузку временного U-Boot (`pyamlboot`) и утилиту `update.exe`.
- **Дамп разделов** — чтение разделов с устройства в файлы `.img`. Таблица
  разделов Yandex Station Max встроена (21 раздел), поддерживается автоопределение
  через UART и ручная вставка лога `amlmmc part 1`.
- **Редактор ENV** — чтение переменных окружения U-Boot с устройства, их разбор
  и правка (разблокировка bootloader: `lock`, `avb2`, `EnableSelinux`, а также
  `serial`, `mac`, яркость, локализация и т.д.) с последующим `setenv` + `saveenv`.
- **Редактор образов** — распаковка и сборка образов Android (ext4 / sparse /
  erofs) напрямую через консольные утилиты MIK из папки `bin/`, либо запуск
  полного GUI MIK.
- **COM-терминал (UART)** — мониторинг вывода U-Boot и загрузки системы в реальном
  времени, сохранение лога. Вывод команд по USB дублируется в терминал даже без
  подключённого UART-преобразователя.
- **Кастомные разделы** — добавление произвольного раздела (например `tee`) с
  собственным образом.
- **Диагностика USB** — поиск устройства Amlogic (`VID_1B8E` `PID_C003`,
  «WorldCup Device») несколькими методами (WMIC, PowerShell, libusb).
- **Автозагрузка зависимостей** — скачивание всех необходимых файлов из GitHub
  одной кнопкой.

---

## Требования

- **Windows 10/11**
- **Python 3.7+**
- Python-пакеты:
  ```
  pip install pyusb pyserial
  ```
  `pyserial` нужен для COM-терминала (опционально). `pyusb` — для загрузки U-Boot
  и USB-диагностики.
- **Драйверы Amlogic USB Burning Tool** (или WinUSB через Zadig) — без них
  устройство не определяется. См. раздел «Драйверы» ниже.
- Для редактора `update.exe` может потребоваться **Microsoft Visual C++ 2010
  Redistributable** (обычно уже установлен в Windows).

## Установка и структура папок

Распакуйте проект **ближе к корню диска**, путь не должен содержать пробелов и
кириллицы (это требование MIK для сборки образов).

```
flasher/
├── gui.py                  ← главный файл, запускается отсюда
├── files/                  ← утилиты (скачиваются автоматически)
│   ├── update.exe          ← утилита прошивки/дампа Amlogic
│   ├── aml_bundle.img      ← U-Boot образ для загрузки в USB-режиме (S905X2)
│   ├── libusb-1.0.dll
│   └── MIK/                ← Multi Image Kitchen (редактор образов)
│       ├── MIK64.exe
│       └── bin/            ← консольные утилиты: simg2img, img2simg,
│                              make_ext4fs, imgextractor, mkfs.erofs …
├── pyamlboot_local/        ← модуль загрузки U-Boot по USB
│   ├── boot.py
│   └── …
└── images-bkp/             ← образы разделов для прошивки (готовите сами)
    ├── boot.img
    ├── system-bkp.img
    └── …
```

### Автоматическая загрузка файлов

При первом запуске программа проверяет наличие компонентов (`update.exe`, MIK,
`pyamlboot_local/`) и предлагает скачать недостающее. Можно вызвать вручную
кнопкой **«📥 Загрузить утилиты»**.

Загрузка работает так:
- весь репозиторий **suddosu/yasta_flasher** скачивается одним zip-архивом —
  всё содержимое `files/` и `pyamlboot_local/` раскладывается по местам
  (гарантированно попадают все файлы, включая `.dll`);
- **MIK** скачивается отдельным zip из репозитория **CryptoNickSoft/MIK** в
  `files/MIK/`.

> ⚠️ Если `pyamlboot_local/` появилась только после скачивания — **перезапустите
> программу**, чтобы модуль корректно подгрузился.

Файлы, которые нужно подготовить **вручную**:
- `files/aml_bundle.img` — U-Boot образ, специфичный для вашего устройства;
- образы разделов в `images-bkp/` — берутся из дампа вашего устройства или
  совместимой прошивки.

## Драйверы

Без драйверов Amlogic устройство не появится в системе. Варианты установки:

1. **USB Burning Tool** — установите официальную утилиту, она поставит драйверы.
   После можно удалить саму программу, драйверы останутся.
2. **Zadig** (`https://zadig.akeo.ie/`) — Options → List All Devices → выберите
   `GX-CHIP` / `Amlogic` (VID `1B8E`, PID `C003`) → установите драйвер **WinUSB**
   или **libusb-win32**.

В программе есть кнопка **«🔍 Диагностика USB»**, которая показывает, видит ли
система устройство, и подсказывает, что делать, если нет.

## Аппаратное подключение

1. Найдите сервисную USB-колодку под резинкой возле радиатора.
2. **Замкните пин 6 на пин 3 (GND)** для входа в USB Boot режим.
3. Подключите USB к ПК.
4. Подайте питание на устройство.

Для UART-мониторинга подключите USB-UART преобразователь (CH340, CP2102, FT232…)
к пинам TX / RX / GND, скорость **115200 8N1**.

## Как пользоваться

### Прошивка

1. Подготовьте образы в `images-bkp/` или укажите их кнопками «Обзор…».
2. Отметьте галочками разделы для прошивки.
3. Нажмите **«🚀 Начать прошивку»**.
4. Программа загрузит U-Boot, прочитает ENV с устройства и **предложит открыть
   редактор ENV**. Прошивка разделов начнётся **только после закрытия редактора**.
5. После применения ENV (`setenv` + `saveenv`) последовательно прошиваются
   выбранные разделы.

> Для получения root на этом устройстве в ENV обычно задают:
> `EnableSelinux=permissive`, `avb2=0`, `lock=10000000`.

### Дамп разделов

1. Нажмите **«💾 Дамп разделов»**.
2. Таблица разделов Yandex Station Max уже загружена — можно дампить сразу.
3. Нажмите **«⚡ Запустить сессию»**, чтобы загрузить U-Boot и (при подключённом
   UART) автоматически считать актуальную таблицу с устройства.
4. Отметьте нужные разделы (есть «Выбрать все / Снять все / Инвертировать»).
5. Укажите папку и нажмите **«💾 Дамп выбранных разделов»**.

Дамп использует `update.exe mread store <раздел> normal <размер> <файл>`, где
размер берётся из таблицы (секторы × 512). Команда `store disprotect key`
снимает защиту чтения с защищённых разделов.

### Редактор образов

1. Нажмите **«📝 Редактор образов»**.
2. Выберите образ — программа определит формат (ext4 / sparse / erofs).
3. **«📂 Распаковать»** — распакует образ в папку (откроется Проводник).
4. Отредактируйте файлы в папке.
5. **«📦 Собрать образ»** — соберёт новый `.img` через `make_ext4fs`.
6. Для сложных случаев (erofs и пр.) — **«🖥 Открыть GUI MIK»**.

> ⚠️ Путь к образу и папке распаковки **не должен содержать пробелов и кириллицы**
> — иначе `make_ext4fs` не соберёт образ. Имена файлов внутри образа тоже не
> должны содержать пробелов.

### COM-терминал

Панель справа. Выберите COM-порт, нажмите «Подключить» — увидите вывод U-Boot и
загрузку системы. Лог можно сохранить в файл.

## Важные предупреждения

- **Не отключайте устройство и не прерывайте процесс прошивки** — есть риск
  превратить устройство в «кирпич».
- Перед дампом/прошивкой делайте **резервную копию** разделов.
- Все образы должны соответствовать именно вашей модели устройства.
- Программа взаимодействует с низкоуровневым загрузчиком — используйте на свой
  страх и риск.

---

## Используемые инструменты

- **update.exe** — Amlogic USB Burning Tool (CLI): прошивка, дамп, команды U-Boot.
- **pyamlboot** — загрузка временного U-Boot по USB.
- **MIK (Multi Image Kitchen)** — `https://github.com/CryptoNickSoft/MIK` —
  распаковка/сборка образов Android из Windows.
- **pyserial** — COM-терминал (UART).
- **pyusb / libusb** — USB-диагностика.
