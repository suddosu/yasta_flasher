# 🔧 Yandex Station Max Flasher - Графический инсталлятор прошивки

Графическое приложение для прошивки Yandex Station Max (Amlogic S905X2) на Windows.

## 📋 Возможности

- ✅ **Автоматическая загрузка утилит** из GitHub (update.exe, DLL библиотеки)
- ✅ **Графический интерфейс** - удобный выбор образов
- ✅ **Выборочная прошивка** - выбирайте какие разделы прошивать
- ✅ **Журнал операций** - отслеживание процесса в реальном времени
- ✅ **Работа с ENV** - автоматическая модификация переменных окружения
- ✅ **Проверка файлов** - валидация наличия всех необходимых компонентов
- ✅ **COM терминал (UART)** - мониторинг вывода устройства в реальном времени
- ✅ **Диагностика USB** - проверка подключения устройства

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install pyamlboot
pip install pyserial  # Для COM терминала (опционально)
```

### 2. Запуск программы

```bash
python bkp-full-gui.py
```

### 3. Первый запуск

При первом запуске программа:
1. Покажет инструкцию по необходимым файлам
2. Автоматически проверит наличие утилит
3. Предложит загрузить недостающие файлы из GitHub

## 🔧 Автоматическая загрузка утилит

Программа автоматически загружает необходимые файлы из официального репозитория:
- **Источник**: https://github.com/khadas/utils/tree/master/aml-flash-tool/tools/windows
- **Файлы**: 
  - `update.exe` - основная утилита прошивки
  - `aml_image_v2_packer` - упаковщик образов (опционально)

**Важно**: Для работы `update.exe` может потребоваться **Microsoft Visual C++ 2010 Redistributable Package**. Обычно он уже установлен в Windows, но если программа не запускается - установите этот пакет.

Загрузка происходит:
1. Автоматически при первом запуске (если файлы отсутствуют)
2. Вручную через кнопку "📥 Загрузить утилиты"
3. Через кнопку "🔍 Проверить наличие файлов"

## 📦 Что нужно подготовить вручную

### 1. aml_bundle.img
U-Boot образ для загрузки в USB режим (специфичен для S905X2).
Должен быть получен вместе с прошивкой устройства.

### 2. Образы разделов (images-bkp/)
Все файлы .img из прошивки вашего устройства:
- boot.img
- dtbo.img
- vbmeta.img
- logo.img
- odm.img
- product.img
- recovery.img
- **system-bkp.img** (самый большой, ~2-3 ГБ)
- vendor.img
- sysrecovery.img

⚠️ **ВАЖНО**: Используйте только образы, соответствующие вашей модели устройства!

## 🔌 Подключение устройства

### Распиновка сервисной колодки

```
          ┌─── GND 
          │┌─── RX  (UART)
          ││┌─── TX  (UART)
          │││
        1234567  
        ◘◘◘◘◘◘◘ 
┌───────┘││  │  
DATA+ ┌───┘├──┘  
      DATA-  │     
            GND
```

### Последовательность подключения:

1. **Подключите USB кабель** к сервисной колодке (находится снизу под резинкой, возле радиатора)
   - DATA+ к пину 1
   - DATA- к пину 7
   - GND к пину 3
   
2. **(Опционально) Подключите USB-UART преобразователь** для мониторинга:
   - TX преобразователя → RX устройства (пин 2)
   - RX преобразователя → TX устройства (пин 3)
   - GND преобразователя → GND устройства (пин 3)
   - Параметры: 115200 бод, 8 бит данных, без чётности, 1 стоп-бит (8N1)

3. **Замкните 6 пин на землю** (3 пин) для активации USB boot режима

4. **Подключите USB к ПК**

5. **Подключите блок питания** к станции

## 💻 Использование программы

### Шаг 1: Выбор образов
- Отметьте чекбоксами разделы, которые нужно прошить
- Используйте кнопку "Обзор..." для выбора других файлов
- Кнопки "✓ Выбрать все" / "✗ Снять все" для быстрого выбора

### Шаг 2: Проверка файлов
Нажмите "🔍 Проверить наличие файлов" чтобы:
- Проверить наличие утилит
- Проверить наличие образов
- Загрузить недостающие утилиты (если нужно)

### Шаг 3: Начало прошивки
1. Нажмите "🚀 Начать прошивку"
2. Подключите устройство согласно инструкции
3. Программа автоматически:
   - Обнаружит устройство
   - Загрузит U-Boot
   - Модифицирует переменные окружения (ENV)
   - Прошьет выбранные разделы

### Шаг 4: Мониторинг
- Следите за журналом операций
- Прогресс-бар показывает активность
- Время прошивки зависит от размера разделов

### Шаг 5: COM терминал (опционально)
1. В правой панели выберите COM порт вашего UART преобразователя
2. Нажмите "Подключить"
3. Вы увидите вывод U-Boot и системы в реальном времени:
   - Загрузка U-Boot
   - Выполнение команд прошивки
   - Загрузка системы после прошивки
4. Кнопка "Сохранить лог" сохранит всё в текстовый файл

**Преимущества COM терминала:**
- Видите что происходит на устройстве
- Полезно для отладки проблем
- Не блокирует процесс прошивки
- Можно подключить/отключить в любой момент

## ⚠️ Важные предупреждения

1. **НЕ отключайте устройство** во время прошивки
2. **НЕ используйте USB-хабы** - подключайте напрямую к ПК
3. **Убедитесь в стабильности питания** - используйте качественный блок питания
4. **Делайте резервные копии** перед прошивкой
5. **Используйте правильные образы** для вашей модели

## 🔍 Что делает программа

### 1. Загрузка U-Boot
Загружает временный загрузчик через USB для доступа к eMMC

### 2. Модификация ENV
Изменяет переменные окружения U-Boot:
```
setenv silent 0      # Включить вывод
setenv lock 10000000 # Разблокировать
setenv avb2 0        # Отключить Android Verified Boot
saveenv              # Сохранить изменения
```

### 3. Прошивка разделов
Последовательно прошивает выбранные разделы:
- Маленькие разделы: несколько секунд
- system: ~10 минут
- sysrecovery: ~10 минут

## 🐛 Решение проблем

### Устройство не определяется
- Установите драйверы Amlogic USB Burning Tool
- Проверьте замыкание 6 пина на землю
- Попробуйте другой USB-порт (желательно USB 2.0)
- Перезагрузите ПК и попробуйте снова

### Ошибка загрузки утилит
- Проверьте подключение к интернету
- Попробуйте загрузить повторно через кнопку "📥 Загрузить утилиты"
- Загрузите файлы вручную из: https://github.com/khadas/utils/tree/master/aml-flash-tool/tools/windows

### update.exe не запускается
- Установите **Microsoft Visual C++ 2010 Redistributable Package**
- Скачать можно с официального сайта Microsoft
- Попробуйте запустить от имени администратора

### Ошибка при прошивке
- Убедитесь, что образы соответствуют устройству
- Проверьте целостность файлов образов
- Попробуйте прошить разделы по отдельности

### pyamlboot не найден
```bash
pip install pyamlboot
```

### pyserial не найден (COM терминал не работает)
```bash
pip install pyserial
```
Если не устанавливать pyserial - программа будет работать, но без COM терминала.

### COM порт не определяется
- Установите драйверы для вашего USB-UART преобразователя:
  - **CH340/CH341**: драйверы с сайта производителя
  - **CP2102/CP2104**: Silicon Labs CP210x USB to UART Bridge VCP Drivers
  - **FT232**: FTDI VCP Drivers
- Проверьте в Диспетчере устройств раздел "Порты (COM и LPT)"
- Попробуйте другой USB порт

### Нет вывода в COM терминале
- Проверьте правильность подключения TX/RX (они должны быть перекрёстно)
- Убедитесь что скорость 115200 (программа использует её автоматически)
- Попробуйте поменять местами TX и RX
- Проверьте что GND подключен

## 📚 Дополнительная информация

### COM терминал (UART)

**Поддерживаемые USB-UART преобразователи:**
- CH340/CH341 (самые распространённые и дешёвые)
- CP2102/CP2104 (Silicon Labs)
- FT232RL/FT232H (FTDI)
- PL2303 (Prolific)
- И любые другие, создающие виртуальный COM порт

**Настройки:**
- Скорость: 115200 бод (автоматически)
- Биты данных: 8
- Чётность: Нет
- Стоп-биты: 1
- Управление потоком: Нет

**Что вы увидите в терминале:**
1. Вывод U-Boot при загрузке
2. Команды прошивки и их результаты
3. Загрузку Android/Linux системы
4. Сообщения ядра (kernel messages)
5. Логи системных сервисов

**Полезно для:**
- Отладки проблем прошивки
- Проверки правильности загрузки
- Диагностики системных ошибок
- Мониторинга процесса в реальном времени

### Логи
Все операции записываются в журнал в реальном времени.
При возникновении ошибок - скопируйте лог для диагностики.

### Время прошивки
- Маленькие разделы (boot, dtbo и т.д.): 5-30 секунд каждый
- system: 8-12 минут
- sysrecovery: 8-12 минут
- **Общее время**: 20-30 минут для полной прошивки

### Особенности S905X2
- Чип поддерживает USB Boot режим
- Требуется замыкание специального пина
- U-Boot загружается в RAM без модификации eMMC
